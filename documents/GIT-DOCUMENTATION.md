# GitHub Documentation
This docuementation is for informate the developer and the members of project on how to use GitHub Project planner.
The planner can be found here: [GitHub Project Planner](https://github.com/orgs/kth-ii1302-vt22-group-3/projects/4/views/2).

## Workflow

### Individual Workflow
1. Go to [Current sprint table](https://github.com/orgs/kth-ii1302-vt22-group-3/projects/4/views/6) .
2. Look at the **No Assignees** category.
3. Find a task which you want to work on.
4. Assign yourself to the task.


### Group Workflow
1. Go to [Project Planner List](https://github.com/orgs/kth-ii1302-vt22-group-3/projects/4/views/2).
2. Add tasks to under **Product Backlog**.
3. Add a story.
  1. Go to [Stories Table](https://github.com/orgs/kth-ii1302-vt22-group-3/projects/4/views/5).
  2. Click on the plus sign on top right at the last column, that will add a new column which can enter the User story name.
  3. Go back to **Project Planner List**.
  4. Add the newly created story.
4. Add points.
5. Add total hours.
6. Repeat 2-5 until no task are left.
7. After adding all task move task to their fitted sprint.


## Project Planner List
**PRIVATE USE**  
The **Project Planner List** tab for entering all the task and user stories to the product backlog. 
A input can by entered directly to a sprint or to a product backlog which will be used for gathering all the tasks in one section and can be distributated through the sprints.

1. Initate the input field in **Product Backlog** section.
2. Is your input a **STORY**/**TASK**? Enter your task with uppercase. `CREATE AN APPLICATION.`
3. Enter a name which describe the work the task will solve.
4. (Optional) Click on the task name to get to a description page where you can list what you want to be done for the task.
5. Change sprint tag.
6. Change story tag.
7. Add points.
8. Add total hours.


## Project Planner Table
**PUBLIC USE**  
This page display the task in each represented sprint. 
This page is not for interactive with but just to show external parties on how we plan our project and what are aimed to be completed on represented sprint.


## Current Sprint List
**PRIVATE USE**  
This page display all task for current sprint group together with Assignees column.
This page is used for easier assign yourself to task.


## Current Sprint Table
**PUBLIC USE**  
This page displays the current sprint and the state of tasks. 


## To Do List
**PRIVATE USE**  
This page display what you as a member of this project are assigned to do. 
This page will mostly have 'drafted' task and 'issue' task. 
- **Drafted** task, are task with a hollow and and striped circle. This task are not linked to a repository.
- **Issue** task, are filled circle with a dot. This task are linked to a repository.


## Stories Table
**PUBLIC USE**  
This page display each User stories and their sub task. 
This page contains all the stories that will build up the test cases we have developed.
